# Q1. 자바스크립트에서 this 키워드의 사용과 그 특성에 대해 설명해 주세요.
<details>
  <summary>(1) 기본 개념</summary>

  * **현재 실행 중인 코드에서의 '주인 객체'를 가리키는 키워드**
  * **this는 "어디서 선언됐냐"가 아니라 "어떻게 호출됐냐"로 결덩됨**
</details>

<details>
  <summary>(2) 상황별 this 값</summary>

  * **1. 일반 함수 호출**
  ```JavaScript
  function test() {
      console.log(this);
  }
  test();

  👉 브라우저: `window`
  👉 strict mode: `undefined`
  ```

  * **2. 객체의 메서드로 호출**
  ```JavaScript
  const obj = {
    name: '홍길동',
    greet() {
      console.log(this.name);
    }
  };
  obj.greet();

  👉 `this === obj`
  👉 점(.) 앞에 있는 객체가 this
  ```

  * **3. 생성자 함수 / 클래스**
  ```JavaScript
  function Person(name) {
      this.name = name;
  }
  const p = new Person("홍길동");

  👉 `this === 새로 생성된 객체`
  ```

  * **4. 화살표 함수⭐**

  ```JavaScript
  const obj = {
      name: "홍길동",
      greet: () => {
          console.log(this.name);
      }
  };

  👉 `this`는 자기 자신이 아니라, 바깥 스코프의 this를 그대로 사용

  // 결과 : undefined
  ```

  * **5. call / apply / bind**
  ```JavaScript
  function greet() {
      console.log(this.name);
  }

  greet.call({ name: "홍길동" });
  ```
</details>

<details>
  <summary>(3) 핵심 정리</summary>

  * this는 **동적 바인딩**
  * **호출 방식에 따라 값이 바뀜**
  * 화살표 함수는 **this를 바꾸지 않고 상위 스코프 것을 그대로 씀**
</details>

# Q2. 렉시컬 스코프(Lexical Scope)의 개념과 그 특성에 대해 설명해 주세요.
<details>
  <summary>(1) 기본 개념</summary>

  * **함수가 "어디서 호출됐냐"가 아니라 "어디서 정의됐냐"로 스코프가 결정되는 것**
  * 즉, **코드 작성 위치(구조)에 따라 변수 접근 범위가 정해짐**
</details>

<details>
  <summary>(2) 예제</summary>

  ```JavaScript
  const x = 1;

  function outer() {
      const x = 2;

      function inner() {
          console.log(x);
      }

    inner();
  }

  outer();

  // 결과 : 2
  // 이유:
  // - inner는 outer 안에서 정의됨
  // - So, outer의 x를 참조
  ```
</details>

<details>
  <summary>(3) 호출 위치는 중요하지 않음</summary>

  ```JavaScript
  const x = 1;

  function outer() {
      const x = 2;

      function inner() {
          console.log(x);
      }

      return inner;
  }

  const fn = outer();
  fn();

  // 결과 : 2
  // 이유
  // - inner는 이미 outer의 스코프를 기억하고 있음
  // - 이게 바로 클로저(Closure)
  ```
</details>

<details>
  <summary>(4) 특징 정리</summary>

  * 스코프는 **정의 시점에 결정됨**
  * 실행 위치와 상관 없음
  * 내부 함수는 외부 함수의 변수 접근 가능
  * 클로저를 만들 수 있음
</details>

<details>
  <summary>(5) this vs 렉시컬 스코프(핵심 비교)</summary>

  |구분|this|렉시컬 스코프|
  |-----|-----|-----|
  |기준|호출 방식|선언 위치|
  |결정 시점|실행 시|정의 시|
  |변경 가능 여부|가능(call, bind 등)|불가능|
  |화살표 함수|상위 this 사용|영향 없음|
</details>

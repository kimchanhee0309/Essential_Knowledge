# Q1. 자바스크립트에서 얕은 복사(Shallow Copy)와 깊은 복사(Deep Copy)에 대해 설명해 주세요.

<details>
  <summary>(1) 얕은 복사(Shallow Copy)</summary>

  * **개념**
    * 👉 **겉만 복사하고, 내부(참조값)는 공유하는 복사**
      * 1 depth까지만 복사
      * 객체 안의 객체는 그대로 같은 주소를 참조
        
  * **예제**
  ```JavaScript
  const original = {
      name: "홍길동",
      address: {
          city: "서울"
      }
  };

  const copy = { ...original };

  copy.address.city = "부산";
  console.log(original.address.city); // 부산

  // 👉 why? address는 같은 객체를 바라보고 있음
  ```

  * **특징**
    * 빠름(참조만 복사)
    * 메모리 효율적
    * But, **의도치 않은 사이드 이펙트 발생 가능**
  
  * **대표적인 얕은 복사 방법**
  ```JavaScript
  Object.assign({}, obj)
  { ...obj }  // 스프레드 문법
  array.slice()
  Array.from()
  ```
</details>

<details>
  <summary>(2) 깊은 복사(Deep Copy)</summary>

  * **개념**
    * 👉**객체 내부까지 전부 새로 복사**
    * 모든 중첩 객체까지 완전히 분리
    * 원본과 완전히 독립
    
  * **예제**
  ```JavaScript
  const original = {
      name: "홍길동",
      address: {
          city: "서울"
      }
  };

  const copy = structuredClone(original);
  copy.address.city = "부산";
  console.log(original.address.city); // 서울
  ```

  * **특징**
    * 완전히 독립적인 데이터
    * 안전함(사이드 이펙트 없음)
    * But, 비용이 큼(성능, 메모리)
      
  * **대표적인 깊은 복사 방법**
    *  **1. `structuredClone()` (⭐️ 최신, 가장 추천)**
      ```JavaScript
      const copy = structuredClone(obj);

      // 장점
      // - 대부분의 타입 지원
      // - 안전함
      ```
    * **2. JSON 방식(제한 있음)**
      ```JavaScript
      const copy = JSON.parse(JSON.stringify(obj));

      // 단점
      // 함수, undefined, Date, Map/Set xxxx
      ```

    * **3. 직접 구현(재귀)**
      ```JavaScript
      function deepCopy(obj) {
          if (typeof obj !== "object" || obj === null) return obj;

          const result = Array.isArray(obj) ? [] : {};

          for (let key in obj) {
            result[key] = deepCopy(obj[key]);
          }

          return result;
      }
      ```
</details>

<details>
  <summary>(3) 핵심 차이</summary>

  |구분|얕은 복사|깊은 복사|
  |-----|-----|-----|
  |복사 범위|1단계|전체|
  |참조 공유|O|X|
  |속도|빠름|느림|
  |안정성|낮음|높음|
</details>

<details>
  <summary>(4) 언제 뭘 써야할까?</summary>

  * **얕은 복사를 써도 되는 경우**
    * 단순 객체(중첩 없음)
    * 불변성 관리 안 중요할 때
    
  * **깊은 복사를 써야 하는 경우**
    * 중첩 객체 있음
    * React 상태 관리
    * 원본 데이터 보호 필요
</details>

# Q2. var, let, const를 서로 비교해 설명해 주세요.

<details>
  <summary>(1) 한 눈에 비교</summary>

  |특징|var|let|const|
|-----|-----|-----|-----|
|재선언|가능|불가능|불가능|
|재할당|가능|가능|불가능|
|스코프|함수|블록|블록|
|호이스팅|있음|있음(TDZ)|있음(TDZ)|
|사용 권장|x|O|OO|
</details>

<details>
  <summary>(2) var</summary>

  * **특징**
    * **함수 스코프**
    * **재선언 가능(위험)**
    * 호이스팅 시 **undefined로 초기화**
      
  * **문제점**
    * 실수를 잡기 어려움
      ```JavaScript
      var x = 1;
      var x = 2; // 에러 없음
      ```
    * 블록이 무시됨
      ```JavaScript
      for (var i = 0; i < 3; i++) {}
      console.log(i); // 3
      ```
</details>

<details>
  <summary>(3) let</summary>

  * **특징**
    * **블록 스코프 `{}`**
    * 재선언 X
    * 재할당 O
      
  * **TDZ(Temporal Dead Zone)**
    ```JavaScript
    console.log(a);  // ReferenceError
    let a = 10;

    // 선언 전에 접근 불가
    ```
  * **블록 스코프**
    ```JavaScript
    if (true) {
      let x = 10;
    }
    console.log(x);  // 에러
    ```
</details>

<details>
  <summary>(4) const</summary>

  * **특징**
    * **블록 스코프**
    * 재선언 x
    * 재할당 x
    * 선언과 동시에 초기화 필요
      
  * **중요한 포인트(헷갈림 주의)**
    * **const는 "값 불변"이 아니라 "참조 불변"**
      ```JavaScript
      const obj = { name: "길동" };
      obj.name = "철수"; // 가능

      // 객체 내부는 변경 가능
      ``` 
</details>

# Q1. 시맨틱 태그
<details>
  <summary>(1) 시맨틱 태그란?</summary>

  * 태그 자체가 의미를 담고 있는 것을 말함
</details>

<details>
  <summary>(2) 시맨틱 태그를 활용하면 좋은 점</summary>

  * **웹 접근성(Accessibility) 개선** : 스크린 리더가 문서 구조를 빠르게 이해하고 효율적으로 탐색할 수 있어 접근서잉 향상됨
  * **SEO(검색엔진 최적화) 향상** : 검색엔진의 이해도가 향상되어 컨텐츠의 핵심을 더 정확히 파악할 수 있어 SEO를 최적화할 수 있음
  * **코드 가독성 & 유지보수 용이** : 코드만 보고도 의미 파악이 가능하며, 구조가 명확하여 수정 및 확장 등 유지보수에 용이
  * 표준을 준수하여 웹 표준에 맞춘 구조이기 때문에 호환성이 향상됨
</details>

# Q2. Cascading
<details>
  <summary>(1) Cascading이란?</summary>

  * 같은 요소에 여러 스타일 규칙이 충돌할 때, 어떤 값이 최종적으로 적용될 지 결정하는 원리
</details>

<details>
  <summary>(2) 우선순위 3가지</summary>

  * **1. 중요도** : 선언 위치와 !important로 우선순위가 달라지며, 가장 강력한 우선순위임
  * **2. 명시도**
    * 선택자가 특정하는 범위가 좁을수록 우선순위가 높으며, 선택자가 얼마나 구체적인가를 따짐
    * 중요도가 같은 경우, '선택자가 얼마나 구체적으로 요소를 지칭했는가?'를 따짐
  * **3. 코드순서**
    * 중요도 및 명시도가 같다면 나중에 선언된 규칙이 적용됨
    * why? Cascading은 위 → 아래로 흘러가기 때문에 가장 나중에 선언된 규칙이 적용됨
</details>

<details>
  <summary>(3) 스타일 적용 우선 순위</summary>
  
  |순위|기준|설명|
|------|---|---|
|1순위|**!important**|속성 뒤에 강제 적용 선언|
|2순위|인라인 스타일|HTML 태그에 직접 **style="..."** 작성|
|3순위|아이디 선택자|**#header**|
|4순위|클래스/속성/가상 클래스|**.box, [type="text"], :hover**|
|5순위|태그 선택자|**div, p**|
|6순위|전체 선택자|*|
|7순위|브라우저 기본 스타일|User Agent Stylesheet(가장 낮음)|
</details>

# Q3. CSS에서의 Box Model
<details>
  <summary>(1) Box Model이란?</summary>

  * 웹 요소가 사각형 박스로 구성되며, Content → Padding → Border → Margin 순으로 겹치는 구조를 뜻함
</details>

<details>
  <summary>(2) 구성요소 및 역할</summary>

  * **1. Content** : 가장 안쪽에 위치, 이미지나 텍스트 등 **실제 정보를 담음**, width/height로 크기가 설정됨
  * **2. Padding** : Content-Border 사이에 위치(**내부 여백** 담당), 배경색은 이 영역까지 적용됨
  * **3. Border** : Padding-Margin 사이에 위치, **경계선(테두리) 역할**을 함
  * **4. Margin** : 테두리 바깥쪽 **외부 여백**에 위치, **요소 간 간격 조절 역할**을 함
</details>

# Q4. DNS(Domain Name System)
<details>
  <summary>(1) DNS(Domain Name System)란?</summary>

  * 숫자로 된 IP 주소를 사람이 읽기 쉬운 이름으로 바꿔주는 일종의 '전화번호부' 서비스를 뜻
</details>

<details>
  <summary>(2) DNS 조회 과정</summary>

  * **1. 사용자가 [주소] 입력**
  * **2. 브라우저/OS 캐시 확인** : 있으면 바로 접속 / 없으면 3단계로
  * **3. DNS 서버(8.8.8.8)에 질의** : "[주소]의 IP가 뭐예요?"
  * **4. DNS 서버가 IP 주소 반환** : "xxx.xxx.xxx.xx 입니다!"
  * **5. 해당 IP의 서버에 접속** : HTML/CSS/JS 받아서 화면 렌더링
</details>

<details>
  <summary>(3) 핵심 개념</summary>

  |개념|설명|비유|
|------|---|---|
|**IP 주소**|서버를 구별하는 고유 숫자 주소|집 도로명 주소|
|**도메인**|IP를 사람이 외우기 쉽게 만든 이름|집에 붙인 별명|
|**DNS 서버**|도메인↔IP 변환 정보를 가진 서버|전화번호부|
|**DNS 캐시**|한 번 조회한 결과를 임시 저장 → 재방문 시 빠르게 접속|자주 가는 식당 번호 외우|
</details>

# Q5. URL(Uniform Resource Locator)
<details>
  <summary>(1) URL(Uniform Resource Locator) 이란?</summary>

  * 웹상에 존재하는 자원이 어디에 있는지 알려주는 일관된 지침서
</details>

<details>
  <summary>(2) 구성 요소</summary>

  * **1. Scheme(프로토콜)** : 통신 방식, 즉 어떤 방식으로 대화할 지 정하는 규약, Https://와 같은 것을 정의
  * **2. Host(Domain)** : 찾아갈 서버의 이름(주소)를 정의하는 곳
  * **3. Path(경로)** : 리소스의 위치, 즉 서버 안의 구체적인 경로를 정의하는 곳
  * **4. Query(매개변수)** : 추가 정보 전달, 즉 서버에게 전달하는 추가 조건을 정의하는 곳
  * **5. Anchor(프래그먼트)** : 페이지 내 위치, 즉 페이지 내 특정 위치로 바로 이동하게 만들어 주는 
</details>
